# 外部APIを発行しているレジシステム — 調査結果

飲食店・小売で使われるレジのうち、**開発者向けに外部APIを公式に提供している**主なシステムをまとめました。社員ポータルから売上データなどを連携する際の候補として参照できます。

---

## 1. 公式API・開発者向けドキュメントが明確なもの

### スマレジ（Smaregi）


| 項目     | 内容                                                                                    |
| ------ | ------------------------------------------------------------------------------------- |
| API    | **スマレジ・プラットフォームAPI**（REST）                                                            |
| ドキュメント | [developers.smaregi.dev](https://developers.smaregi.dev/platform-api-reference/)（日本語） |
| 対象プラン  | プレミアム以上（プランにより利用可能機能が異なる）                                                             |
| 主な機能   | 商品・在庫、顧客・取引・売上、発注・入出荷、プッシュ通知など                                                        |
| 認証     | AppAccessToken / UserAccessToken                                                      |
| 備考     | 国内約47,000店舗利用。アプリマーケットでサードパーティアプリも提供。                                                 |


### Loyverse


| 項目     | 内容                                                                                                                     |
| ------ | ---------------------------------------------------------------------------------------------------------------------- |
| API    | **Loyverse POS API**（REST）                                                                                             |
| ドキュメント | [loyverse.com](https://loyverse.com/jp/loyverse-pos-api)、[Read the Docs](https://loyverse.readthedocs.io/)（英語、一部日本語対応） |
| 主な機能   | レシート（売上）取得、会計・在庫・顧客データの同期、カスタムレポート用の売上エクスポート                                                                           |
| 業態     | 飲食店・カフェ・バー等を想定                                                                                                         |
| 備考     | 日本向けサイト・ヘルプあり。開発者コミュニティ（Loyverse Town）あり。                                                                              |


### Square


| 項目     | 内容                                                                      |
| ------ | ----------------------------------------------------------------------- |
| API    | **Square API**（決済・POS・在庫・顧客など）                                          |
| ドキュメント | [developer.squareup.com/jp](https://developer.squareup.com/jp/ja)（日本向け） |
| 主な機能   | 決済、POS連携（他社POSアプリとSquare Reader連携）、売上・レポート、在庫・顧客                        |
| 備考     | 日本でも「Build with Square」でAPI開放。POS APIで他社レジとSquare決済の連携が可能。              |


### STORES（ストアウェア）


| 項目     | 内容                                                                                |
| ------ | --------------------------------------------------------------------------------- |
| API    | **STORES API**（EC・レジ連携）                                                           |
| ドキュメント | [retail-api-docs](https://heyinc.github.io/retail-api-docs/)（GitHub公開）、要開発パートナー申込 |
| 対象     | STORESスタンダードプラン                                                                   |
| 主な機能   | オーダー取得（STORESレジのオーダー含む）、アイテム・在庫・入金情報など                                            |
| 備考     | ネットショップ＋実店舗レジ。飲食専用というよりEC・小売向け。                                                   |

## 2. API連携の案内はあるが、形態が異なるもの

### ユビレジ（Ubiregi）


| 項目  | 内容                                             |
| --- | ---------------------------------------------- |
| API | 管理画面から「APIアクセストークン」を発行して連携可能                   |
| 注意  | **Ubiregi API は 2025年12月16日で公開終了予定**。新規開発は非推奨。 |
| 備考  | CData API Driver 等によるノーコード連携の紹介あり。             |


### ORANGE POS / poscube / blaynレジ など

- **ORANGE POS**: 「ORANGE DB」経由で管理システムと接続する形態の案内あり。公式REST APIの有無は要確認。
- **poscube**: テーブル・注文紐付けやオーダー送信などの連携機能の案内あり。開発者向け公開APIの有無は要確認。
- **blaynレジ**: スタンダード／プレミアムでAPI連携対応と案内。詳細は各社へ問い合わせが必要。

---

## 3. 今回の調査で「公式の開発者API」が見つからなかったもの

- **USENレジ（u-regi）**: 売上実績等は管理画面からCSV出力。freee・TableCheck等の決まった連携先向け連携はあるが、一般公開の開発者APIドキュメントは見つかっていない。自社システム連携は問い合わせが必要。

---

## 4. 選び方の目安（社員ポータル連携の場合）

- **すでにUSENレジを使用している**  
→ 現状はCSV取り込みでの連携が現実的。APIの有無・仕様はUSENへ要問い合わせ。
- **これからレジを選ぶ／変更を検討している**  
→ 自社開発のポータルから売上などを取りたい場合は、**スマレジ・Loyverse・Square** のように公式APIとドキュメントが公開されている製品を候補にすると、連携しやすい。
- **飲食店特化でAPI連携を重視**  
→ **Loyverse** は飲食・F&B向けを前面に出しており、レシート（売上）APIが明確。**スマレジ** は業態を問わず店舗数が多く、API機能が豊富。
